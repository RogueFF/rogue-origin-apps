## üìù Documentation Standards

When adding new documentation:

1. **Choose the right directory**:
   - Technical docs ‚Üí `/technical`
   - Design specs ‚Üí `/design`
   - Implementation plans ‚Üí `/plans`
   - User guides ‚Üí `/guides`
   - Session notes ‚Üí `/sessions`

2. **Use clear naming**: `YYYY-MM-DD-descriptive-name.md` for dated docs

3. **Include frontmatter** (if applicable):
   ```markdown
   # Document Title

   > Brief description

   **Status**: Draft | In Progress | Complete
   **Last Updated**: YYYY-MM-DD
   ```

4. **Cross-reference related docs**: Link to other relevant documentation
